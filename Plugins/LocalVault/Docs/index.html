LocalVault Reference Manual

Welcome to the official documentation for LocalVault. Designed specifically for Unreal Engine 5.7+, LocalVault is an essential utility plugin that bridges the gap between Epic's cloud-based Fab ecosystem and your local development environment. It provides unprecedented control over your digital assets through instant offline search, custom metadata tagging, and robust, true-offline backup capabilities.

Offline Backups

Extract cached Fab assets directly to a secure, permanent directory on your hard drive.

Persistent Tagging

Apply custom metadata to your library. Search and filter instantly without internet latency.

Zero-API Security

No Epic login required. We natively scan the engine's local cache, ensuring your credentials remain yours.

The LocalVault Philosophy

With the transition from the legacy Unreal Engine Marketplace to the unified Fab ecosystem, the way developers access assets has fundamentally changed. Epic's current infrastructure relies heavily on "on-demand" cloud access, streaming assets into hidden, obscured cache directories (FFabAssetsCache).

While convenient for discovery, this creates a critical vulnerability for independent studios: Asset Permanence. What happens if an asset is delisted? What if you are working offline on a laptop? What if you need to migrate your massive library to a secure internal NAS?

LocalVault was engineered to solve this. It operates on the principle that if you bought it, you should hold the raw files. It acts as a middleman—reading the temporary cache Epic creates, allowing you to organize it, and then permanently backing it up to a standardized directory format.

Initial Configuration

To ensure your assets are backed up to the correct location, you must configure the plugin immediately after installation.

  1. Open your Unreal Engine project.
  2. Navigate to Edit > Project Settings.
  3. Scroll down the left pane to the Plugins section and select LocalVault.
  4. Locate the property: Local Vault Storage Path.
  5. Click the ... button and select an empty, high-capacity directory on your machine (e.g., D:\UnrealVaultArchive).

Drive Space Warning

Unreal Engine assets, particularly environment megapacks and MetaHumans, can consume hundreds of gigabytes. Ensure the drive you select for your Local Vault Storage Path has ample free space. Do not use your primary OS drive (C:\) if space is limited.

Once configured, you can access the main tool window via the Editor Menu: Window > Asset Management > LocalVault Browser.

The Sync Engine (Zero-API)

Epic Games does not currently provide a public REST API for developers to securely query a user's Fab inventory without complex OAuth integration. Instead of asking you to log in to our tool, LocalVault uses a Zero-API Strategy.

When you click the Sync with Epic button in the LocalVault Browser, the following occurs synchronously:

  • The plugin queries the engine's internal C++ module: FFabAssetsCache::GetCachedAssets().
  • It reads the directory structure of the hidden local cache generated by the Unreal Editor.
  • It extracts the folder names to construct human-readable asset titles.
  • It registers these items in the LocalVault JSON Database, marking them with a system tag.

Crucial Workflow Step

Because LocalVault reads the local cache, it can only "see" assets that you have physically downloaded through the Unreal Editor at least once. If you buy an item on the Fab website but do not download it in the Editor, LocalVault will not know you own it.

Library Organization

The LocalVault Browser is split into two primary areas: the Metadata Sidebar (left) and the Asset Grid/List (right).

Applying Tags

You can apply persistent, offline tags to any asset to group them by project, genre, or utility.

  1. Select one or multiple assets in the main view (use Ctrl+Click or Shift+Click).
  2. In the bottom left corner of the sidebar, type your desired tag into the text input field.
  3. Click the + button or press Enter.

The tag will instantly appear as a Pill under the asset's title. Selecting the tag from the sidebar list will instantly filter your entire view to show only assets with that tag. Because the database is stored entirely locally in memory, this search is instantaneous—no loading spinners.

True Offline Backup

This is the core functionality of the plugin. The temporary cache used by the Epic Editor can be wiped, corrupted, or automatically managed by the engine. To secure your assets, you must perform a Bulk Download.

Execution

  1. Select the assets you wish to back up in the LocalVault Browser.
  2. Click the large blue Download Assets button in the top toolbar.
  3. Check your Output Log.

Under the hood, LocalVault executes a deep, multi-threaded directory copy. It takes the raw files from the obscure AppData cache path and cleanly duplicates them into your configured LocalVaultStoragePath, creating a neat folder using the Asset's readable title.

// Example Directory Structure after Backup
D:\UnrealVaultArchive\
├── Cyberpunk Character Pack\
│   ├── Content\
│   └── Config\
├── Stylized Water Shader\
│   └── Content\
└── Ultimate FPS Weapons\
    └── Content\

Complete Ownership

Once backed up, these folders are yours. You can migrate them to an external hard drive, zip them for cold storage, or drag-and-drop their Content folders directly into new Unreal projects without ever opening the Epic Launcher.

Technical Architecture

For developers and studio IT managers, understanding where LocalVault stores its data is critical for version control and workspace migration.

  • Database Location: [ProjectRoot]/Saved/LocalVault/Database.json
  • Format: Standard JSON. Can be manually edited or migrated.
  • Settings Location: [ProjectRoot]/Config/DefaultEditorPerProjectUserSettings.ini
  • Engine Dependency: Requires Unreal Engine 5.7+ due to reliance on the internal Fab C++ module.

Troubleshooting & FAQ

Why does the Output Log say "0 assets found" when I click Sync?
This means your FFabAssetsCache is completely empty. LocalVault operates by scanning the assets you have downloaded via the native Editor.

Fix: Open the official Epic Fab window in UE5, download at least one asset to your local machine, and click Sync again.
Why did the backup fail with a copy error?
This almost exclusively occurs due to Windows path-length limitations (the 260 character MAX_PATH limit) or insufficient drive space.

Fix: Ensure your configured Storage Path is close to the root of a drive (e.g., D:\Vault instead of D:\MyFiles\Projects\UE5\Plugins\Archives\Vault). Check your available disk space.
Can I share my tagged database with my team?
Yes! If you are using Perforce or Git, simply ensure that Saved/LocalVault/Database.json is not ignored by your version control (Unreal ignores the Saved folder by default). Add a specific inclusion rule for the LocalVault subfolder. When your team pulls the file, they will see your custom tags.
